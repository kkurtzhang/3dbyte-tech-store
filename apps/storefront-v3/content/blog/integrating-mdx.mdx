# Integrating MDX in Next.js

MDX lets you use JSX components inside Markdown. This creates powerful possibilities for content creation.

## Why MDX?

- **Rich components**: Use React components in your content
- **Maintainable**: Keep content and UI logic together
- **Type-safe**: Full TypeScript support
- **Flexible**: Works with existing MDX tools

## Basic Setup

First, install the required packages:

```bash
pnpm add @next/mdx @mdx-js/loader @mdx-js/react next-mdx-remote
```

## Creating MDX Content

Here's a simple MDX file:

```mdx
---
title: "My Post"
date: "2026-02-08"
---

# Hello MDX

This is **bold** and *italic* text.

<Link href="/about">About Us</Link>
```

## Custom Components

You can customize how MDX renders different elements:

```tsx
import { MDXRemote } from 'next-mdx-remote/rsc';

const components = {
  h1: ({ children }) => <h1 className="text-3xl">{children}</h1>,
  // ... other components
};

export function MDXContent({ source }) {
  return (
    <MDXRemote source={source} components={components} />
  );
}
```

## Syntax Highlighting

Add code highlighting with rehype-highlight:

```tsx
import rehypeHighlight from 'rehype-highlight';

const options = {
  mdxOptions: {
    rehypePlugins: [rehypeHighlight],
  },
};
```
